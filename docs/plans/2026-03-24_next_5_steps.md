## Step 4 — SMC Trader (execution agent)

**Goal**: An execution agent that reads confirmed SMC theses and opens/closes
MT5 orders accordingly.

**Scope** (`src/heuristic_mt5_bridge/smc_desk/trader/`):
- `trader.py` — `SmcTraderService`:
  - Reads `ThesisStore.list_symbols()`, loads latest thesis per symbol
  - Evaluates `entry_signal`, `stop_loss_pips`, `take_profit_pips` from thesis
  - Opens pending order via `CoreRuntimeService.connector` (MT5 API)
  - Checks open positions against thesis validity; closes on thesis flip
- `risk.py` — lot-size calculator (account balance × risk_percent / SL_pips)
- Wire into `SmcDeskService` when `SMC_TRADER_ENABLED=true`
- New env vars: `SMC_TRADER_ENABLED`, `SMC_TRADER_RISK_PERCENT` (default 1.0)
- New table: `smc_trade_log` (broker-partitioned, records every order action)
- Tests: mock MT5 connector, assert no order placed when thesis confidence < threshold

**Key constraints**:
- Never open a second position in the same symbol if one is already open
- Hard max 2% risk per trade; reject if `risk_percent > 2`
